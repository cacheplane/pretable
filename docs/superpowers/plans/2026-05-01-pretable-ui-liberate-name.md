# Liberate `@pretable/ui` Name — Implementation Plan (PR 1 of 5)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Free the `@pretable/ui` package name by relocating its currently-used contents (Nav, Footer, cool-slate `tokens.css`, slim `marketing-components.css`) into the two apps that consume them, deleting dead code (Receipt, Callout, CodeBlock — none imported by any app), and removing the `packages/ui/` directory entirely.

**Architecture:** Pure relocation. Two React components and two CSS files move from `packages/ui/src/` into `apps/website/app/components/`, `apps/website/app/styles/`, `apps/bench/src/components/`, and `apps/bench/src/styles/`. Nav and Footer are duplicated into both apps (per spec D1 X — each app owns its own chrome). All app imports updated. The `@pretable/ui` workspace dependency dropped from both apps. `packages/ui/` directory deleted. No new code; no visual change at runtime.

**Tech Stack:** TypeScript, React 19, Vitest, Next.js 15 app router (apps/website), Vite (apps/bench), pnpm workspace.

**Spec:** [docs/superpowers/specs/2026-05-01-pretable-theming-architecture-design.md](../specs/2026-05-01-pretable-theming-architecture-design.md) — see PR decomposition table, PR 1.

**Starting state assumption:** Clean working tree on branch `feat/theming-design` at HEAD `aa230a1` (or later). Worktree should be set up via `superpowers:using-git-worktrees` before Task 1.

## Decisions baked in

- **YAGNI deletion of dead components.** Receipt, Callout, CodeBlock (and their tests, and their CSS classes) are not imported by any app. Verified via `grep -rn 'from "@pretable/ui"' apps/ packages/ --include="*.tsx" --include="*.mdx"`. They are deleted, not relocated. If anyone wants them back, git history at commit `64fe888` (or earlier) preserves them.
- **No CodeBlock collision.** `apps/website/app/components/CodeBlock.tsx` exists (a server-side shiki async component), but the marketing CodeBlock from `packages/ui/` is not being moved — so no rename is needed.
- **Nav and Footer duplicated, not shared.** Per spec D1 X. Each app owns its own copy going forward and may diverge.
- **`marketing-components.css` is slim.** Only `.pt-nav`, `.pt-footer`, and any selectors those rules depend on (e.g., `.pt-nav-link`, `.pt-nav-cta`, `.pt-footer-link` if present). The original `components.css` has rules for receipt, callout, code-block — all dropped during the slim. The exact rules included are determined by reading `packages/ui/src/components.css` and copying every rule whose selector starts with `.pt-nav` or `.pt-footer` (case-sensitive).
- **`tokens.css` copied verbatim.** Cool-slate tokens stay exactly as they are. They become the website's local brand tokens (and bench's local copy). No content changes.
- **No tests deleted from the apps.** Tests for Nav and Footer move with the components and continue running in their new homes.

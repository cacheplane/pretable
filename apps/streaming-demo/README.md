# @pretable/app-streaming-demo

Replay-style demo app showing the Pretable streaming adapter feeding an OpenAI-Responses-format stream into a grid with a realtime pipeline inspector. Plays a ~30 s table fill (500 stock rows) followed by ~90 s of continuous price updates, with play / pause / scrub controls.

## Development

```
pnpm dev      # Vite dev server with HMR
pnpm build    # production build
pnpm test     # unit tests (vitest)
```

The app ships with checked-in recordings at `src/recordings/phase1.jsonl` and `src/recordings/phase2.jsonl`. The runtime is fully self-contained — no network calls.

## Regenerating recordings

### Canonical path: synthesize phase 1 + regenerate phase 2

```
pnpm make-dev-fixture
pnpm generate-phase2
```

Both scripts are deterministic (seeded PRNG with `0xC0FFEE`), produce byte-identical output across runs, and require no network access.

`make-dev-fixture.ts` synthesizes 500 rows of curated stock data (real ticker symbols like AAPL, MSFT, GOOGL — variants generated by suffixing) and emits them as OpenAI Responses-format SSE deltas with realistic chunk sizes (5–15 chars) and timing (~30 s total span, modeled after captured gpt-5 streams).

`generate-phase2.ts` reads `phase1.jsonl`, parses the seed rows, and writes 90 s of price-update batches at ~500 patches/sec to `phase2.jsonl`.

### Why synthesized?

Real LLM captures (gpt-5, gpt-4o) reliably failed to produce 500 varied rows in a single response — they truncated, repeated values, wrapped output in markdown, or refused. The synthesis path produces visually richer, more reliable demo data while still exercising the streaming adapter, JSON parser, and grid surface end-to-end exactly as a real capture would.

### Real OpenAI capture (optional)

```
export OPENAI_API_KEY=sk-...
pnpm capture
pnpm generate-phase2   # re-derive phase 2 from the new phase 1
```

The `capture-phase1.ts` script remains in the repo for anyone who wants to attempt a real LLM capture. Set `OPENAI_MODEL=...` to override the default (`gpt-5`) and `STREAMING_DEMO_ROW_COUNT=...` to override the row count (default `500`). Be aware: in current testing, models tend to either truncate output, lose variety in numeric fields, or wrap output in markdown — the synthesis path is the canonical demo recording.
